Rust offers you not only the advantages (and challenges) of a statically and strongly typed language but also takes one step further with ownership and borrowing and a really strict compiler

Typescript, for example, can give you the taste of a statically and typed language, however, in reality, it's really hard to compare Rust with Typescript when comparing language features

Before we go and briefly compare the Typescript syntax with the Rust syntax, here's a really brief explanation of Ownership and Borrowing.

Ownership and Borrowing

Always have these 3 rules of Ownership in mind:

  1. Each value in Rust has a variable that’s called its owner

  2. There can only be one owner at a time

  3. When the owner goes out of scope, the value will be dropped 

fn main() {
    // variable declared, s1 has ownership, rule 1
    let s1 = String::from("world");

    // value is moved, s1 ceases to exist
    // s2 now has the ownership of the value rule 2
    let s2 = s1;

    // compile error, s1 value is moved, you need to use s2 instead
    format!("Hello {}", s1);

} // end of the function, s2 is dropped from the memory, rule 3

That's a design choice, Rust by design will always prefer to move your values rather than copying or cloning them. However, copies does take memory space and sometimes you just need to borrow the value 

fn main() {
    let s1 = String::from("world");

    // the ampersand tells the program that you don't want to move the value
    // but rather use the same pointer in memory
    let s2 = &s1;

    println!("Hello {}", s1); // no error
    println!("Hello {}", s2); // no error
}

Ownership and Borrowing

Beware: the compiler won't allow you to make changes on borrowed values. You have to create new values FROM the borrowed value.

A brief look at Rust's syntax

Side by side, the basic syntax is not that far from TypeScript (or Javascript). And that's how far I will go when comparing both of them. I wont' be mentioning Traits or Structs here but think of them as Interfaces and Types, respectively. 

// constants and variables
// type: number, inferred
const MEANING_OF_LIFE = 42

// can cast from other types
const NOT_MEANING_OF_LIFE: boolean = !MEANING_OF_LIFE

// string type inferred
let helloWorld = "Hello World!"

// functions


function greet (name: string): string {

	return `Hello ${name}`
}

greet("Bob")

// loop
for (let entry of [1, 2, 3]) {
	console.log(entry); // 1, 2, 3
}

// control flow
if (entry > 10) {
    console.log('more than 10')
} else if (entry >= 5) {
    console.log('more than or equal 5')
} else {
    console.log('less than 5')
}
